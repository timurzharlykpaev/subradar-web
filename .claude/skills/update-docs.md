# Skill: Update Documentation

When making changes to web code, update the relevant documentation:

1. **New page/screen** -> Update `docs/WEB_SCREENS.md`
2. **New/changed API hook** -> Update `agent_docs/api-contracts.md`
3. **New/changed type** -> Update `agent_docs/types.md`
4. **New/changed UI component** -> Update `agent_docs/design-system.md`
5. **Changed deployment config** -> Update `agent_docs/deployment.md`
6. **New i18n keys** -> Update `agent_docs/i18n.md`
7. **Completed MVP item** -> Check off in `PROGRESS.md`

Shared docs (PRODUCT_OVERVIEW, DOMAIN_MODEL, API_CONTRACTS, BILLING_RULES, AI_BEHAVIOR, STATE_RULES) canonical source is subradar-backend. Update there first if the change is cross-repo.
